LMP API Guide

This page describes how to use the LMP compile/decompile API. The API has no server: all compilation runs in your browser using JavaScript. There is no backend to call; the site is static and the conversion happens on your machine.

How it works

The API is a plug-and-play URL. You open a minimal page with your data and arguments in the URL (in the hash fragment). That page loads the LMP compiler in the browser, runs the conversion, triggers a file download, and can close automatically. Scripts, bookmarklets, or other apps can open this URL programmatically (e.g. from Tampermonkey or a link).

Main API endpoint

The conversion is done by a minimal subpage that shows no text on success—only a download (and optional auto-close). That page is:

api/run.html

Open it with hash parameters (see below). With valid parameters it will compile or decompile, start the download, and if close=1 is set it will close the window. With invalid or missing parameters it displays only a JSON error.

Parameters

Pass parameters in the URL hash as key=value pairs separated by &. Values should be encoded (e.g. encodeURIComponent for filename). LMP text and MIDI binary must be base64-encoded.

ParameterDirectionMeaning
dirbothlmp2midi or midi2lmp
lmplmp2midiBase64-encoded UTF-8 LMP text
midimidi2lmpBase64-encoded MIDI binary
filenamebothDownload filename (optional)
looselmp2midi1 = use loose mode
preferSpnmidi2lmp0 or 1 (default 1)
precisionmidi2lmpBeat decimal places (default 3)
closeboth1 = close window after download

Example

To compile LMP to MIDI and download with auto-close, open:

https://zunaidfarouque.github.io/LMP-Specification/api/run.html#dir=lmp2midi&lmp=<base64>&filename=out.mid&loose=1&close=1

Replace <base64> with your LMP text encoded as base64 (UTF-8). In JavaScript you can use: btoa(unescape(encodeURIComponent(lmpText))).

Errors and warnings

If the runner hits an error (e.g. invalid base64 or compile failure), it shows only a JSON object in the page, e.g. {"error": "..."}. If loose mode reports warnings, it may show {"warnings": ["..."]}. There is no other text or UI on the runner page.
