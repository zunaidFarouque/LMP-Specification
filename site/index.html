What is LMP?

The Lean Musical Protocol (LMP) is a stateful, Space-Separated Value (SSV) grid format designed to act as the perfect intermediary language between AI models and binary MIDI data.

Unlike traditional music notation or standard MIDI files—which are illegible or mathematically confusing to modern Byte-Pair Encoding (BPE) tokenizers—LMP is built from the ground up for bidirectional LLM interaction. It allows AI models to "read" existing MIDI files, understand their musical context, and "write" new, mathematically precise compositions without context window bloat.

The Bidirectional Pipeline

  1. MIDI → LMP → AI: An LMP parser converts binary .mid files into compressed LMP text. LLMs can read, analyze, and understand melodies, rhythms, or multitrack arrangements using minimal tokens.
  2. AI → LMP → MIDI: The LLM generates or modifies music by writing LMP syntax. The parser compiles this deterministic text into a standard, playable .mid file.

Why LMP?

When tasking LLMs with music generation or analysis, standard data formats fail:

  • JSON/XML: Massive token bloat from structural repetition.
  • CSV: Commas disrupt BPE tokenization and degrade mathematical reasoning.
  • Standard MIDI (Delta-Time): LLMs struggle with cumulative arithmetic, causing rhythmic drift.

The LMP Solution:

  • BPE-Optimized Whitespace: SSV grid compresses contiguous whitespace.
  • Absolute Floating-Point Grid: Beat placements (e.g. 1.0, 2.5) eliminate delta-time arithmetic errors.
  • Stateful DRY Architecture: Cascading fallbacks and track defaults reduce redundancy.

Use Cases

  • Autonomous Agent Workflows: Analyze audio/MIDI, rewrite arrangements, output new files.
  • Algorithmic Composition: Rapidly prototype multi-track sequences, legato, or custom drum maps (e.g. Tabla bols).
  • Dataset Generation: Compress MIDI libraries into efficient LMP text for fine-tuning LLMs.
